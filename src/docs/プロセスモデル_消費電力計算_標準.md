# プロセスモデル：消費電力計算（標準）

## 概要

本ドキュメントでは、暖房・冷房消費電力の標準計算プロセスを詳細に記載します。

暖房と冷房は同じプロセスで処理されるため、統合して記述します。

**記法:** 
- `{H|C}` : 暖房はH、冷房はC
- `{温度|温度・湿度}` : 暖房は温度のみ、冷房は温度・湿度の両方

**注意:** 負荷計算（calc_heating_load/calc_cooling_load）は`pyhees`既存ライブラリをそのまま使用しています。

**各モデルの特徴:**

| type | 名称 | 特徴 |
|------|------|------|
| type1 | ダクト式セントラル | 機器仕様による性能曲線 |
| type2 | RAC現行省エネ法 | JIS C 9612基準のCOP |
| type3 | RAC潜熱評価 | 潜熱負荷を詳細評価 |
| type4 | 電中研モデル | R係数による特性曲線 |

| `Theta_hs_in_d_t` | ℃ | 吸込温度 | 8760時間 |
| `Theta_ex_d_t` | ℃ | 外気温度 | 8760時間 |

## ステップ1: 送風機設計風量の決定

```mermaid
flowchart TD
```

**計算式:**
- `V_fan_dsgn = V_fan_rtd × 係数`（詳細は仕様書参照）

## ステップ2: 未処理負荷と風量・温湿度の計算

### ステップ概要

このステップでは主要な関数 `calc_Q_UT_A()` を呼び出します。

- 初期化・気象データ読込
- 床下温度
- 各区画の設定温度・湿度
- 各区画の供給風量
- 給気温度・湿度
- 吹出温度・湿度
- 吸込温度・湿度
- 未処理負荷

## ステップ3: 熱源機平均能力の計算

```mermaid
flowchart LR
    Start[熱源機能力計算] --> Input[入力:<br/>- Theta_hs_out_d_t<br/>- Theta_hs_in_d_t<br/>- V_hs_supply_d_t<br/>- X_hs_out_d_t, X_hs_in_d_t 冷房のみ]

    style Calc fill:#e1ffe1
```

**計算式:**
- 暖房: `q_hs_H_d_t = get_q_hs_H_d_t(...)`
- 冷房: `q_hs_CS_d_t, q_hs_CL_d_t = get_q_hs_C_d_t_2(...)`

## ステップ4: 送風機消費電力の計算

```mermaid
flowchart TD
    Start[送風機消費電力] --> Type{計算モデル判定}
    
    Type -->|type1: ダクト式<br/>type2: RAC現行<br/>type4: 電中研| Standard[標準式<br/>get_E_E_fan_d_t]
    
    Type -->|type3: RAC潜熱| Latent[潜熱評価式<br/>latent_load.get_E_E_fan_d_t]
    
    Standard --> Formula[E_E_fan = f<br/>P_fan_rtd,<br/>V_hs_vent,<br/>V_hs_supply,<br/>V_hs_dsgn,<br/>q_hs,<br/>f_SFP]
    
    Latent --> FormulaL[E_E_fan = f<br/>V_hs_vent,<br/>q_hs,<br/>f_SFP]
    
    Formula --> Result[E_E_fan_d_t]
    FormulaL --> Result
    
    style Type fill:#ffffcc
```

**標準式の概要:**
```
E_E_fan_d_t = (V_hs_supply_d_t / V_hs_dsgn)^3 × P_fan_rtd / 1000
            または
              V_hs_vent_d_t × f_SFP / 1000
```
（詳細は条件分岐あり）
